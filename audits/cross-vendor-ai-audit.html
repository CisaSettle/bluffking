我们的信任模型不是"请相信我们",而是三件可以自己核的东西:

Our trust model is not "trust us." It's three things you can check yourself:

  1. 代码开源 —— 发牌加密全在这个仓库,你能直接读。
  2. Open source — the dealing crypto is in this repo; read it directly.
  3. 跨厂商 AI 双审 —— 一家的模型(Claude)写,另一家公司的模型(OpenAI Codex, gpt-5.5)独立审,达成共识(OPEN 清单为空)才发布。不是自审。
  4. Cross-vendor AI dual-audit — one company's model (Claude) writes; a different company's model (OpenAI Codex, gpt-5.5) independently reviews; nothing ships until they reach consensus (empty OPEN list). Not self-review.
  5. 可运行的离线验证器 —— 仓库里带了 demo transcript 和 KAT 向量;自己跑 phase4_blind_demo 看本地服务端盲发,或对内置 transcript 跑 mp-verify 得到 Sound / DevMock 判定。
  6. A runnable offline verifier — the repo ships demo transcripts + KAT vectors; run the phase4_blind_demo example to watch the server-blind deal locally, or run mp-verify on an included transcript for a Sound / DevMock verdict.

审了什么(engine-blind 服务端盲发)What was audited (engine-blind server-blind dealing)

mental-poker/src/crypto_real/ 的 n-of-n 服务端盲发发牌:分布式密钥生成(DKG)、可验证重加密洗牌、门限 ElGamal / Chaum–Pedersen 解密、Ed25519 签名的摊牌见证。它在生产环境为全真人 engine-blind 牌桌上线运行(ADR-070,2026-06-23)——服务器看不到你已弃牌或摊牌前的底牌。

mental-poker/src/crypto_real/ — the n-of-n server-blind deal: distributed key generation (DKG), verifiable re-encryption shuffle, threshold ElGamal / Chaum–Pedersen decryption, and an Ed25519-signed showdown attestation. It runs in production for all-human engine-blind tables (ADR-070, 2026-06-23) — the server cannot see your folded or pre-showdown hole cards.

审计记录(GA 门槛)Audit record (the GA gate)

  • CONSENSUS ADR-070 上线门槛 = 一次干净的独立跨厂商 Codex(gpt-5.5)审计:5 轮设计共识(Codex thread 019eef65,OPEN 清单为空)+ 一次干净的实现审计(thread 019ef015,OPEN 清单为空)。
  • CONSENSUS The ADR-070 GA gate = one clean independent cross-vendor Codex (gpt-5.5) audit: a 5-round design consensus (Codex thread 019eef65, empty OPEN) + a clean implementation audit (thread 019ef015, empty OPEN).
  • reveal-gate 谱系 ADR-076 → 077 → 078:跨厂商审计发现并砍掉了一个不成立的"first-mover"引导(会重新打开底牌泄露),换成 Ed25519 DKG-绑定的签名摊牌见证(ADR-078,Accepted,binding)。审计抓真问题的实证。
  • Reveal-gate lineage ADR-076 → 077 → 078: the cross-vendor audit found and killed an unsound "first-mover" bootstrap (it re-opened the hole-card leak) and replaced it with an Ed25519 DKG-bound signed showdown attestation (ADR-078, Accepted, binding). Evidence the audit catches real problems.
  • 生产形态双人验收:服务器持有 0 / 52 张牌的解密份额;ADR-078 的 MpShowdownAttest 签名见证在真实连线上触发。
  • Prod-shaped 2-human acceptance: the server held 0 of 52 decryption shares; the ADR-078 MpShowdownAttest signed attestation fired over the live wire.
  • 不可能性也被诚实记录:原设计里的 (t,n) 门限 + owner-blinding 机制被跨厂商审计判定为密码学上不可能,当场砍掉(ADR-070 §2 D5)——我们不藏砍掉的东西。
  • Even the impossibility is on record: the original (t,n)-threshold + owner-blinding idea was ruled cryptographically impossible by the cross-vendor audit and dropped (ADR-070 §2 D5) — we don't hide what got cut.

诚实边界(我们说什么、不说什么)The honest ceiling (what we do and don't claim)

范围,直说:服务端盲发只适用于全真人 engine-blind 牌桌(≥2 真人、无 bot)。在这些桌上,服务器看不到你已弃牌或摊牌前的底牌;摊牌牌面按扑克规则公开。默认发牌、任何带 bot 的牌桌、实验性的 co-shuffle、mobile 原生路径都不是服务端盲发。这是 play-money 学习产品。

Scope, stated plainly: server-blind applies only to all-human engine-blind tables (≥2 humans, no bots). On those, the server can't see your folded or pre-showdown hole cards; showdown cards are public by the rules of poker. Default dealing, any table with bots, the experimental co-shuffle toggle, and native-mobile play are not server-blind. It's a play-money study product.

你自己怎么核How to verify it yourself

  1. 读加密代码: mental-poker/src/crypto_real/(dkg / shuffle / decrypt / ed25519_signer)。
  2. Read the crypto: mental-poker/src/crypto_real/ (dkg / shuffle / decrypt / ed25519_signer).
  3. 跑验证器: cargo run -p mental-poker --example phase4_blind_demo(本地看真实盲发),以及对仓库内置 demo/fixture transcript 跑 mp-verify 得到 Sound / DevMock 判定(mock 路径 fail-closed,不会谎报公平)。
  4. Run it: cargo run -p mental-poker --example phase4_blind_demo (the real server-blind deal, locally), and run mp-verify on the repo's demo/fixture transcripts for a Sound / DevMock verdict (the mock path fails closed; it won't claim fairness for mock crypto).
  5. 读决策记录: ADR-070 / 076 / 077 / 078(私有仓库的架构决策;本页把可公开的结论摘录于此)。
  6. Read the decision records: ADR-070 / 076 / 077 / 078 (architecture decisions in the private repo; the public-safe conclusions are excerpted here).
